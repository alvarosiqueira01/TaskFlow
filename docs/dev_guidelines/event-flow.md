
## Event Lifecycle & Publication

The backend uses an event-driven architecture via Amazon EventBridge. The lifecycle of an event is as follows:

1. **Use Case:** Executes business logic and modifies state.


2. **Domain Event:** A core event (e.g., `TaskCreated`) is generated by the Domain layer.


3. **Application Event:** The use case maps the domain event to an application-level contract.
4. **EventBridge:** The Infrastructure layer publishes the event to the AWS event bus.


5. **Notification Service:** An independent microservice consumes the event.


6. **Email/Notification:** The final output is delivered to the user.
